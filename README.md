# mysql-datatypes

Это заготовка для домашней работы по типам данных в MySQL. Сейчас здесь только простой Docker-контур: один контейнер MySQL, именованный volume для данных и папка `snapshots`, куда позже будем складывать скриншоты с доказательствами работы.

## Что уже подготовлено

- MySQL запускается через `compose.yaml`;
- используется официальный образ `mysql:9.7.0`;
- настройки можно менять через `.env`;
- команды вынесены в `Makefile`;
- SQL-файлы для первичной инициализации можно класть в `docker/mysql/init`;
- скриншоты для README будем хранить в `snapshots`.

Почему версия указана явно, а не просто `latest`? Так меньше сюрпризов: проект поднимает свежую версию MySQL, но не начинает вести себя иначе после очередного обновления образа.

## Быстрый старт

Если нужны свои пароли или порт, скопируй `.env.example` в `.env` и поменяй значения. Можно и не копировать: в `compose.yaml` уже есть спокойные значения по умолчанию.

Запуск:

```bash
make up
```

Проверка статуса:

```bash
make ps
```

Открыть консоль MySQL:

```bash
make mysql
```

Пароль по умолчанию: `root_password`.

Остановить контейнер:

```bash
make down
```

Полностью удалить контейнеры вместе с volume:

```bash
make clean
```

Важно: `make clean` удалит данные MySQL. Для учебного стенда это удобно, но перед запуском лучше всё же подумать: база точно не нужна?

## Подключение

По умолчанию MySQL доступен на `localhost:3306`.

Параметры из `.env.example`:

- база данных: `mysql_datatypes`;
- пользователь: `otus`;
- пароль пользователя: `otus_password`;
- пароль `root`: `root_password`.

С хоста можно подключиться так:

```bash
mysql -h 127.0.0.1 -P 3306 -u otus -p mysql_datatypes
```

## Структура

```text
.
├── compose.yaml
├── docker/
│   └── mysql/
│       └── init/
├── snapshots/
├── .env.example
├── .gitignore
├── Makefile
└── README.md
```

Папка `docker/mysql/init` пригодится позже: MySQL автоматически выполнит SQL-файлы из неё при первом создании volume. Примечательно, что повторный запуск уже созданной базы эти файлы не переигрывает. Это нормальное поведение контейнера.
